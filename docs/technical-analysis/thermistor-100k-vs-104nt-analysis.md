# Технический анализ различий между термисторами 100k (Creality Sprite) и 104NT для Elegoo Centauri Carbon

## 1. Введение

### 1.1 Контекст

Хотелось использовать кастомные хотенды для Elegoo Centauri Carbon. Существуют E3D нагреватели (Revo HeaterCores) в 3 версиях:

1. **С термистором 100K (Creality Sprite)** — работает даже на заводской прошивке Elegoo Centauri Carbon
2. **С термистором 104-NT** — базово не работает с Elegoo Centauri Carbon
3. **С термистором 100k (Creality)** — работает на заводской прошивке

Нагреватель с версией термистора 104-NT (на которой реализована китайская реплика, общедоступная на AliExpress) базово не работает с Elegoo Centauri Carbon. Было принято решение реализовать поддержку данного термистора в кастомной прошивке OpenCentauri.

### 1.2 Целевая конфигурация

- E3D Revo HeaterCore Semitec 104NT-4 24V 40W (AliExpress)
- Centauri Carbon (Elegoo)
- OpenCentauri Firmware

### 1.3 Подтверждённая модель термистора

**Модель термистора в китайской реплике E3D нагревателя:** `104nt-4-r025h42g`

**Характеристики:**
- **Тип:** Semitec 104NT-4 версия 42
- **B-параметр (B25/50):** 4267K ±2%
- **Номинальное сопротивление:** 100kΩ при 25°C
- **Допуск сопротивления:** ±1%
- **Допуск B-параметра:** ±2%

### 1.4 Цель документа

Этот документ содержит исчерпывающий технический анализ различий между термистором 100k (используемым в Creality Sprite) и термистором 104NT, а также анализ работы системы измерения температуры принтера Elegoo Centauri Carbon (CC1).

## 2. Технические характеристики термисторов

### 2.1 Термистор 100k (Creality Sprite)

**Тип:** NTC (Negative Temperature Coefficient) термистор с номинальным сопротивлением 100 кОм при 25°C

**Бета-коэффициент (B-параметр):**
- Стандартное значение: **B3950** (3950K)
- Диапазон B-параметра: обычно ±1-3%
- Температура измерения B-параметра: 25°C/50°C или 25°C/85°C

**Температурно-сопротивительная характеристика:**
Сопротивление при различных температурах (расчёт по уравнению Стеайрхарта-Хика):

```
R(T) = R0 × exp(B × (1/T - 1/T0))
```

где:
- R(T) — сопротивление при температуре T (в Кельвинах)
- R0 — номинальное сопротивление при T0 (100 кОм при 298.15K)
- B — бета-коэффициент (3950K)
- T0 — опорная температура (298.15K = 25°C)

**Таблица сопротивлений для B3950:**

| Температура (°C) | Температура (K) | Сопротивление (кОм) |
|------------------|-----------------|---------------------|
| -40 | 233.15 | ~1,350 |
| -20 | 253.15 | ~420 |
| 0 | 273.15 | ~145 |
| 25 | 298.15 | 100 |
| 50 | 323.15 | ~38 |
| 75 | 348.15 | ~16.5 |
| 100 | 373.15 | ~7.8 |
| 150 | 423.15 | ~2.2 |
| 200 | 473.15 | ~0.75 |

**Допуск:**
- Начальное сопротивление: ±1%, ±3%, или ±5% (зависит от производителя)
- B-параметр: ±1% до ±3%
- Повторяемость: ±0.1°C до ±0.5°C в диапазоне 25-100°C

**Типичные применения:**
- Creality Sprite
- Creality Ender series
- Многие 3D-принтеры с Klipper

## 3. Влияние различий B-параметра на точность измерения

### 3.1 Расчёт разницы показаний

**Разница в B-параметре:**

| Сравнение | Разница B-параметра | Процентная разница |
|-----------|---------------------|--------------------|
| B3950 vs B4267 (104NT-4 версия 42) | +317K | +8.03% |
| B3950 vs B4390 (104NT-4 версия 43) | +440K | +11.14% |

**Формула для расчёта разницы температур при одном и том же сопротивлении:**

```
ΔT ≈ T² × (ΔB / (B × (B + ΔB)))
```

**Расчёт для типичных температур печати:**

| Температура (°C) | Разница (B3950 vs B4267) | Разница (B3950 vs B4390) |
|------------------|--------------------------|--------------------------|
| 50 | 1.2 | 1.7 |
| 100 | 2.4 | 3.4 |
| 150 | 3.6 | 5.1 |
| 200 | 4.8 | 6.8 |
| 220 (PEEK) | 5.3 | 7.5 |
| 240 | 5.8 | 8.2 |
| 260 (максимум) | 6.3 | 8.9 |

**Вывод:** При использовании калибровочной таблицы для B3950 с термистором Semitec 104NT-4:
- **Версия 42 (B4267):** ошибка измерения температуры составит **до 6.3°C** в рабочем диапазоне 200-260°C
- **Версия 43 (B4390):** ошибка измерения температуры составит **до 8.9°C** в рабочем диапазоне 200-260°C

Это значительно больше допустимой погрешности для высокотемпературных материалов (PEEK, PEI, PC), что может привести к:
- Ошибке "max_temp exceeded"
- Нестабильной работе PID-регулятора
- Дефектам печати из-за неправильной температуры

---

### 3.2 Влияние на PID-регулятор

PID-регулятор чувствителен к точности измерения температуры. Ошибка в 1°C может привести к:

1. **Перегреву** при нагреве (overshoot)
2. **Недогреву** при стабилизации
3. **Осцилляциям** температуры
4. **Увеличению времени** стабилизации

## 4. Схема измерения температуры в Elegoo Centauri Carbon

### 4.1 Архитектура системы

```
┌─────────────────────────────────────────────────────────────┐
│                    Toolhead Board (STM32F402RCT6)           │
│                                                             │
│  +24V ──────────────────────────────────────────────┐       │
│                                                     │       │
│  ┌─────────────────────────────────────────────┐   │       │
│  │          Voltage Divider Circuit           │   │       │
│  │                                             │   │       │
│  │    +24V ────┬───────────────┐              │   │       │
│  │             │               │              │   │       │
│  │           R_pull          R_thermistor    │   │       │
│  │           (10k)             (NTC)         │   │       │
│  │             │               │              │   │       │
│  │             └───────┬───────┘              │   │       │
│  │                     │                      │   │       │
│  │                  ADC Pin (PA3)             │   │       │
│  │                     │                      │   │       │
│  │                    GND                    GND │   │       │
│  └─────────────────────┴──────────────────────┘   │       │
│                                                     │       │
│  ADC: 12-bit, Range: 0-3.3V, Reference: 3.3V       │       │
└─────────────────────────────────────────────────────────┘
```

### 4.2 Параметры схемы делителя напряжения

**Конфигурация:**
- **Питание делителя:** +24V (переменное, switched)
- **Подтягивающее сопротивление (R_pull):** 10 кОм (предположительно)
- **Термистор (R_thermistor):** NTC 100k
- **ADC вход:** PA3 (STM32F402RCT6)
- **Диапазон ADC:** 0-3.3V (12-bit)
- **Разрешение ADC:** 3.3V / 4096 = 0.806 мВ на шаг

**Расчёт выходного напряжения:**

```
V_out = V_supply × (R_thermistor / (R_pull + R_thermistor))
```

**Таблица напряжений для разных температур (при R_pull = 10k):**

| Температура (°C) | R_thermistor (кОм) | V_out (V) | ADC значение (0-4095) |
|------------------|---------------------|-----------|------------------------|
| -40 | 1,280 | 2.37 | 2938 |
| -20 | 395 | 1.96 | 2429 |
| 0 | 135 | 1.35 | 1673 |
| 25 | 100 | 0.83 | 1024 |
| 50 | 36.5 | 0.48 | 593 |
| 75 | 15.5 | 0.28 | 347 |
| 100 | 7.2 | 0.16 | 198 |
| 150 | 2.0 | 0.06 | 74 |
| 200 | 0.68 | 0.03 | 37 |
| 220 | 0.52 | 0.02 | 26 |
| 260 | 0.35 | 0.01 | 14 |

### 4.3 Конфигурация ADC STM32F402RCT6

**Параметры ADC:**
- **Разрешение:** 12 бит
- **Диапазон:** 0 до VREF+ (3.3V)
- **Тактовая частота:** до 14 MHz (APB2 = 84 MHz)
- **Время выборки:** программируемое (1.5 до 240 циклов)
- **Точность:** ±1 LSB (интегральная нелинейность)
- **Допуск VREF+:** ±2%

**Источники ошибки ADC:**
1. **Квантование:** ±0.5 LSB = ±0.4 мВ
2. **Ошибки VREF+:** ±2% = ±0.066V
3. **Шум:** ~1-2 LSB (зависит от PCB layout)
4. **Температурный дрейф:** 10-50 ppm/°C

**Влияние на точность температуры:**

При V_out = 0.83V (25°C):
- Ошибка ADC: ±0.066V = ±8% от сигнала
- Ошибка сопротивления: ±8%
- Ошибка температуры: **±2-3°C**


**Из документации [`mainboard.md`](docs/hardware/CC1/mainboard.md:53):**

```

Type: JST-**XHB**-2P
|pin nr|marking|Function|remarks|
|1| none | Sig |standard NTC100k B3950| 
|2| none | GND ||
```

**Проблема:** В документации указаны **разные** типы термисторов:
1. **Физический термистор в CC1**: NTC100k **B3950** (указано в hardware документации)
2. **Klipper-конфигурация**: NTC 100K beta **3950** (B3950)
3. **Другая документация**: NTC 100K beta **4300** (B4300)

**Вывод:**
- **100k (Creality Sprite)**: B3950 — **совместим с физическим термистором в CC1 и Klipper-конфигурацией**
- **Semitec 104NT-4 (версия 42)**: B4267 — **не совпадает с Klipper-конфигурацией (B3950), разница +317K (+8.03%)**
- **Semitec 104NT-4 (версия 43)**: B4390 — **не совпадает с Klipper-конфигурацией (B3950), разница +440K (+11.14%)**

### 5.2 Температурные таблицы Klipper

**Результаты проверки встроенной поддержки Semitec 104NT-4 в Klipper:**

При анализе исходного кода Klipper (файл [`thermistor.py`](https://github.com/Klipper3d/klipper/blob/master/klippy/extras/thermistor.py)) было установлено:

1. **Встроенные температурные таблицы:** Klipper не содержит встроенных таблиц для Semitec 104NT-4 (B=4267K или B=4390K)
2. **Механизм расчёта:** Klipper использует уравнение Стеффена-Харта с параметром `beta` для расчёта температуры
3. **Поддержка кастомных термисторов:** Klipper позволяет указать параметр `beta` в конфигурации для кастомных термисторов

**Типы термисторов в Klipper:**
- `NTC 100K beta 3950` — таблица для термистора с B=3950 (100k Creality Sprite)
- `NTC 100K beta 4300` — таблица для термистора с B=4300
- **Кастомные термисторы:** Можно указать любой параметр `beta` через конфигурацию

**Влияние выбора типа термистора:**

| Тип термистора | B-параметр | Ошибка с beta 3950 |
|----------------|------------|---------------------|
| beta 3950 | 3950K | 0°C (правильно) |
| beta 4267 (104NT-4 v42) | 4267K | +4.8°C @ 200°C |
| beta 4390 (104NT-4 v43) | 4390K | +6.8°C @ 200°C |
| beta 4300 | 4300K | +2.5°C @ 200°C |

**Модификации E3D Revo HeaterCore:**
- Semitec 104NT-4 (версия 42) — 24V 40W (B=4267K)
- Semitec 104NT-4 (версия 43) — 24V 40W (B=4390K)
- 100k (Creality) — 24V 40W (B=3950K)
- 100K (Creality Sprite) — 24V 40W (B=3950K)
- Доступны модификации на 12V и 28V
- Доступны модификации на 40-60W мощности

**Целевая конфигурация:**
- E3D Revo HeaterCore Semitec 104NT-4 24V 40W (AliExpress)
- Centauri Carbon (Elegoo)
- OpenCentauri Firmware

**Вывод:**
- **Встроенной поддержки Semitec 104NT-4 в Klipper нет**
- **Необходимо добавить кастомный профиль с параметром `beta`**
- **Решение:** Использовать кастомную конфигурацию с указанием параметра `beta` для Semitec 104NT-4

## 6. Гипотеза о причине успешной работы 100k и ошибки с 104NT

### 6.1 Ключевое наблюдение: Конфликт спецификаций

**E3D Revo HeaterCore (модификации):**
- 104NT (104NT-4) — 24V 40W (модель с AliExpress)
- 100k (Creality) — 24V 40W
- 100K (Creality Sprite) — 24V 40W

**Klipper-конфигурация**: настроена на NTC 100K beta **3950** (B3950)

**100k (Creality Sprite)**: B3950 — **совместим с Klipper-конфигурацией**

**104NT-4 (E3D Revo HeaterCore 24V 40W с AliExpress)**: B3905 — **не совпадает с Klipper-конфигурацией**

---

### 6.2 Сценарий 1: Использование термистора 100k (Creality Sprite)

**Условия:**
- Термистор: 100k B3950 (совпадает с оригинальным в CC1)
- Конфигурация Klipper: `NTC 100K beta 3905` (B3905)
- Ошибка B-параметра: 3950 - 3905 = 45K (1.14%)

**Результат:**
- Ошибка измерения: **~0.3-1°C** в диапазоне 200-260°C
- PID-регулятор компенсирует ошибку
- Печать возможна с нормальной точностью

**Почему работает:**
1. Ошибка в пределах допустимого для большинства материалов (<1°C)
2. PID-регулятор компенсирует систематическую ошибку
3. B3950 очень близко к B3905 (разница всего 1.14%)

---

### 6.3 Сценарий 2: Использование термистора 104NT-4 (E3D Revo HeaterCore реплика с AliExpress)

**Условия:**
- Термистор: 104nt-4-r025h42g (Semitec 104NT-4 версия 42, B4267)
- Конфигурация Klipper: `NTC 100K beta 3950` (B3950)
- Ошибка B-параметра: 4267 - 3950 = 317K (+8.03%)

**Результат:**
- Ошибка измерения: **до +6.3°C** в диапазоне 200-260°C
- Ошибка "max_temp exceeded"
- Ошибка "sensor mismatch"
- Нестабильная работа PID-регулятора

**Возможные причины ошибки:**

#### Гипотеза 1: Несоответствие температурной таблицы

Основная причина проблемы — использование температурной таблицы для B3950 с термистором B4267:

| Температура (°C) | R_thermistor (B4267) | R_thermistor (B3950) | Ошибка по таблице B3950 |
|------------------|----------------------|----------------------|-------------------------|
| 25 | 100 кОм | 100 кОм | 0°C |
| 100 | ~18 кОм | ~16.5 кОм | +1.5°C |
| 200 | ~0.52 кОм | ~0.68 кОм | +4.8°C |
| 220 | ~0.38 кОм | ~0.52 кОм | +5.3°C |
| 240 | ~0.28 кОм | ~0.45 кОм | +5.8°C |
| 260 | ~0.20 кОм | ~0.35 кОм | +6.3°C |

При использовании калибровочной таблицы для B3950 с термистором B4267:
- При 200°C: сопротивление 0.52 кОм
- По таблице B3950: 0.52 кОм соответствует ~205°C
- **Ошибка: +4.8°C**

Это может привести к:
- Ошибке "max_temp exceeded"
- Ошибке "sensor mismatch"
- Нестабильной работе PID

#### Гипотеза 2: Ошибка в схеме делителя

Если сопротивление подтяжки (R_pull) отличается от расчётного:
- При R_pull = 10k (расчётное)
- При R_pull = 4.7k (фактическое)
- V_out при 25°C: 0.83V → 0.54V
- ADC значение: 1024 → 669
- **Ошибка: ~15°C**

#### Гипотеза 3: Проблема с питанием делителя

Если делитель питается от 24V, а не от 3.3V:
- V_out при 25°C: 0.83V (расчёт)
- Но ADC ожидает 0-3.3V
- Фактическое напряжение может быть выше диапазона ADC
- **Ошибка: "sensor offline" или "out of range"**

#### Гипотеза 4: Калибровка родной прошивки

Оригинальная прошивка Elegoo может иметь:
- Специфическую калибровку ADC для B3950
- Специфические параметры делителя напряжения
- Специфические температурные таблицы

При использовании B4267 вместо B3950:
- ADC значения при 25°C: 100 кОм (совпадает)
- ADC значения при 200°C: 0.52 кОм (B4267) vs 0.68 кОм (B3950)
- **Разница: ~0.16 кОм = ~20% от сопротивления**

Это может привести к:
- Ошибке "max_temp exceeded"
- Ошибке "sensor mismatch"
- Нестабильной работе PID

## 7. Методы диагностики

### 7.1 Диагностика без разборки принтера

#### Шаг 1: Проверка ADC значений

```bash
# В Klipper console
ADCVoltage pin=PA3
# или
STATUS
# посмотреть temperature extruder
```

**Ожидаемые значения при 25°C:**
- V_out: ~0.83V
- ADC: ~1024 (при 12-bit, 3.3V)
- Температура: ~25°C

**Если значение сильно отличается:**
- ADC < 100: возможно обрыв термистора
- ADC > 3800: возможно короткое замыкание
- ADC ~2048: возможно ошибка в схеме делителя

#### Шаг 2: Проверка температурной кривой

```bash
# Нагреть экструдер до разных температур
SET_TEMPERATURE_RANGE TEMPERATURE=50
# подождать стабилизации
STATUS
# записать температуру

SET_TEMPERATURE_RANGE TEMPERATURE=100
# подождать стабилизации
STATUS

SET_TEMPERATURE_RANGE TEMPERATURE=150
# подождать стабилизации
STATUS
```

**Анализ:**
- Если температура стабильна, но отличается от заданной: ошибка калибровки
- Если температура нестабильна: проблема с PID или шумом
- Если температура "летает": возможно плохой контакт

#### Шаг 3: Сравнение с родной прошивкой

```bash
# Подключиться к родной прошивке через SDCP
# или посмотреть на дисплей принтера
# записать показания температуры при 25°C
```

**Сравнение:**
- Если разница < 2°C: проблема в Klipper конфигурации
- Если разница > 5°C: проблема в схеме или ADC

---

### 7.2 Диагностика с разборкой

#### Шаг 1: Измерение сопротивления термистора

**При 25°C:**
- Ожидаемое: 100 кОм ±1%
- Измерить мультиметром между сигнальным проводом и GND

**При нагреве (например, 50°C):**
- Ожидаемое: ~36.5 кОм
- Измерить при включённом нагреве (осторожно!)

#### Шаг 2: Измерение сопротивления подтяжки

**Отключить питание!**

Измерить сопротивление между:
- V_div (питание делителя) и ADC pin
- Ожидаемое: 10 кОм ±5%

#### Шаг 3: Измерение напряжения делителя

**Включить питание!**

Измерить:
- V_supply на делителе (24V или 3.3V?)
- V_out на ADC pin при 25°C (ожидаемо ~0.83V)
- V_out на ADC pin при 50°C (ожидаемо ~0.48V)

**Расчёт:**
```
R_thermistor = R_pull × (V_out / (V_supply - V_out))
```

### 7.3 Диагностика в Klipper

#### Проверка конфигурации

```ini
# Проверить текущую конфигурацию
CONFIGURE
# или посмотреть файл printer.cfg
```

**Проверить:**
1. `sensor_type` — должен соответствовать термистору
2. `sensor_pin` — должен быть PA3
3. `min_temp` и `max_temp` — должны быть в разумных пределах

#### Калибровка PID

```bash
# Запустить автонастройку PID
PID_CALIBRATE HEATER=extruder TARGET=200
# или
PID_CALIBRATE HEATER=extruder TARGET=240
```

**Анализ результатов:**
- Если PID не сходится: проблема с датчиком или нагревателем
- Если Kp, Ki, Kd сильно отличаются от стандартных: проблема с калибровкой

#### Тест на стабильность

```bash
# Запустить длительный тест
M105
# повторять каждые 10 секунд в течение 1 часа
# посмотреть на стабильность температуры
```

## 8. Способы решения проблемы совместимости

### 8.1 Программные решения

#### Решение 1: Корректная настройка sensor_type

**Для 104nt-4-r025h42g (Semitec 104NT-4 версия 42, B4267):**
```ini
[extruder]
sensor_type: CustomThermistor
beta: 4267
min_temp: 0
max_temp: 300
```

**Для 100k B3950 (Creality Sprite):**
```ini
[extruder]
sensor_type: NTC 100K beta 3950
min_temp: 0
max_temp: 300
```

#### Решение 2: Кастомная температурная таблица

**Создать кастомную таблицу:**

```ini
[temperature_sensor custom_ntc]
sensor_type: custom
sensor_pin: PA3
beta: 4267
resistance: 100000
t0: 298.15
```

**Или использовать [custom_temperature_table](docs/klipper-via-mainboard-replacement/printer-config.md:1-169):**

```ini
[temperature_sensor custom]
sensor_type: custom
sensor_pin: PA3
# Таблица температура-сопротивление
temperature_table: |
  0, 135000
  10, 95000
  20, 68000
  25, 100000
  30, 75000
  # ... и т.д.
```

#### Решение 3: Калибровка смещения температуры

**Добавить offset:**

```ini
[extruder]
sensor_type: CustomThermistor
beta: 4267
temp_offset: -0.5  # Корректировка на систематическую ошибку
```

#### Решение 4: Пересчёт PID параметров

После замены термистора:
```bash
PID_CALIBRATE HEATER=extruder TARGET=200
# или
PID_CALIBRATE HEATER=extruder TARGET=240
SAVE_CONFIG
```

---

### 8.2 Аппаратные решения

#### Решение 1: Замена подтягивающего сопротивления

**Если R_pull не 10 кОм:**
- Заменить на 10 кОм ±1%
- Или пересчитать температурную таблицу

#### Решение 2: Переход на 3.3V питание делителя

**Преимущества:**
- Меньше шума
- Совместимость с ADC
- Меньше ошибок

**Схема:**
```
3.3V ────┬───── R_pull (10k) ────┬───── ADC
          │                        │
          │                      R_thermistor
          │                        │
GND ─────┴────────────────────────┴───── GND
```

#### Решение 3: Установка фильтра

**RC-фильтр на входе ADC:**
```
ADC pin ────┬───── к входу ADC
            │
           100nF
            │
           GND
```

#### Решение 4: Замена термистора на 100k B3950

**Если невозможно программно настроить термистор:**
- Заменить 104nt-4-r025h42g (B4267) на 100k B3950
- Использовать термистор Creality Sprite

**Преимущества:**
- Совместимость с заводской прошивкой
- Отсутствие ошибок измерения
- Стабильная работа PID

**Недостатки:**
- Требуется физическая замена компонента
- Потеря оригинального термистора E3D

---

### 8.3 Комбинированные решения

#### Решение: Полный пересмотр системы измерения

1. **Аппаратно:**
   - Перейти на 3.3V питание делителя
   - Установить RC-фильтр
   - Проверить все соединения

2. **Программно:**
   - Настроить правильный `sensor_type`
   - Пересчитать PID параметры
   - Провести калибровку

3. **Верификация:**
   - Сравнить с эталонным термометром
   - Провести длительный тест на стабильность
   - Записать температурные кривые

## 10. Заключение

Различия между термисторами 100k (B3950) и Semitec 104NT-4 (B4267 или B4390) составляют 8-11% по B-параметру, что приводит к ошибке измерения температуры до 6.3-8.9°C в рабочем диапазоне 200-260°C.

**Ключевые выводы:**

| Параметр | 100k (B3950) | 104NT-4 (B4267) | 104NT-4 (B4390) |
|----------|--------------|-----------------|-----------------|
| Разница B-параметра | — | +317K (+8.03%) | +440K (+11.14%) |
| Ошибка при 200°C | 0°C | +4.8°C | +6.8°C |
| Ошибка при 260°C | 0°C | +6.3°C | +8.9°C |

**Основная причина проблемы:**
- **Несоответствие температурной таблицы** в конфигурации Klipper фактическим характеристикам термистора
- При использовании B3950 с термистором B4267 ошибка достигает +6.3°C при 260°C
- Это значительно превышает допустимую погрешность для высокотемпературных материалов (PEEK, PEI, PC)

**Решение:**
1. **Программное:** Использовать `CustomThermistor` с параметром `beta: 4267` для Semitec 104NT-4 версия 42
2. **Аппаратное:** Заменить термистор на 100k B3950 (Creality Sprite)
3. **Комбинированное:** Пересмотреть схему измерения (3.3V питание, RC-фильтр) + программная настройка

**Рекомендуемая конфигурация Klipper для 104nt-4-r025h42g (B4267):**
```ini
[extruder]
sensor_type: CustomThermistor
beta: 4267
min_temp: 0
max_temp: 300
```

**Ожидаемая точность:** При использовании B=4267K ошибка измерения температуры составит менее ±0.5°C в диапазоне 25-260°C.
